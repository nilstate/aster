# automaton

`automaton` is the public repo where `runx` proves it can be useful without
becoming vague or reckless.

The point is not silent autonomy.

The point is visible, bounded, reviewable automation:

- issues can become draft PRs
- docs can be refreshed through a governed lane
- PRs can receive maintainer comments backed by receipts
- new skill ideas can become concrete proposals instead of loose notes

This repo is intentionally small so evaluators can inspect the full loop:
inputs, workflow trigger, runx packet, scafld artifacts, git diff, and final PR
or comment.
